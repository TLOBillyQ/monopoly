## 工作计划


先通过搜索与引用分析得到冗余清单，再逐项处理。处理顺序为：明确替代 → 更新引用 → 删除/合并 → 运行回归测试。对外入口优先保留兼容薄壳，内部工具或测试入口优先删除。所有改动需要记录在“意外与发现”和“产物与备注”中，便于追溯。


## 具体步骤


1) 盘点冗余/兼容/多套实现。
   - 使用搜索命令：
       rg -n "Runtime|Adapter|EventHandlers|GameEvents" Manager Library Globals
       rg -n "__init\.lua" Manager
       rg -n "deprecated|compat|legacy" -g"*.lua"
   - 建立清单：标注“旧入口 / 重复实现 / 兼容层 / 临时桥接”。

2) 逐项处理清单。
   - 若有新入口替代：更新 `require` 路径后删除旧实现。
   - 若仍被外部脚本调用：保留薄壳转发，并添加简短注释标明替代入口。
   - 若重复实现互相冲突：统一到一处，并删除另一处实现。

3) 清理未使用入口。
   - 对 `rg` 无引用的模块，确认是否被 `require` 动态拼接；若无动态引用，直接删除。
   - 删除后再次搜索确认无引用残留。

4) 校验差异。
   - 检查 `git status` 与关键路径，确保无误删核心实现。
