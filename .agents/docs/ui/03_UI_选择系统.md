# UI：选择系统

## 路由总览

按 `choice.kind` 分流：

1. `market_buy` -> `黑市屏`
2. `remote_dice_value` -> `遥控骰子屏`
3. `item_target_player` -> `玩家选择屏`
4. `roadblock_target` / `demolish_target` -> `位置选择屏`
5. `landing_optional_effect` / `land_optional_effect` 且 options 全属于 `{buy_land, upgrade_land}` -> `建筑升级屏`
6. 其他 kind -> `位置选择屏`

## 玩家选择屏

节点：

- `玩家选择_标题`
- `玩家选择_副标题`
- `玩家选择_槽位1..3`
- `取消按钮`

规则：

- options 按顺序映射到 `槽位1..3`，超出截断。
- 点击任一槽位直接提交对应选项（`choice_select`）。
- `取消按钮` 触发 `choice_cancel`。

## 位置选择屏

节点：

- `位置_副标题`
- `位置_放置文本`
- `位置前1..3`
- `位置后1..3`
- `位置脚下`
- `取消按钮`

规则：

- 静态容量 7（前 3 + 后 3 + 脚下 1）。
- 超出容量按原顺序截断。
- `位置脚下` 仅当 option label 包含“脚下”或“当前位置”时启用。
- 点击任一候选按钮直接提交对应选项（`choice_select`）。
- `取消按钮` 触发取消。

## 遥控骰子屏

节点：

- `遥控骰子_标题`
- `遥控骰子_正文`
- `遥控骰子_选项_01..06`
- `遥控骰子_取消`

规则：

- 点击任一点数按钮立即提交对应 option。
- `遥控骰子_取消` 触发 `choice_cancel`。

## 建筑升级屏

节点：

- `建筑升级_标题`
- `建筑升级_文本`
- `建筑升级_确定按钮`
- `建筑升级_取消`
- （视觉节点）`建筑升级_标题背景`、`建筑升级_弹窗背景`

规则：

- 仅承接地产可选操作（`buy_land` / `upgrade_land`）。
- 标题规则：
  - `buy_land` -> `购买地块`
  - `upgrade_land` -> `加盖建筑`
- `建筑升级_文本` 显示 `choice.body`（为空则显示空字符串）。
- `建筑升级_确定按钮` 提交当前选项。
- `建筑升级_取消` 触发 `choice_cancel`。
- 若出现非地产可选项，回退到 `位置选择屏`。
