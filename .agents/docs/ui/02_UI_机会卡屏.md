# UI：机会卡屏（V2）

## 当前状态与目标状态

- 当前状态：代码仍按旧弹窗节点（`弹窗屏`、`弹窗标题`、`弹窗确认`）组织。
- 目标状态：所有 `push_popup` 统一渲染到 `机会卡屏`。

## 画布与节点

画布：`机会卡屏`

| 节点名 | 类型 | 用途 |
|---|---|---|
| `机会卡_标题` | ELabel | 弹窗标题 |
| `请输入文字` | ELabel | 弹窗正文 |
| `机会卡_图片` | EImage | 卡图/插图 |
| `取消按钮` | EButton | **确认/继续**（本屏语义） |

## 展示规则

`push_popup(payload)` 目标行为：

1. 切换到 `机会卡屏`。
2. 写入标题到 `机会卡_标题`。
3. 写入正文到 `请输入文字`。
4. 若有 `image_ref/image_key`，显示并更新 `机会卡_图片`；无图则隐藏并重置为空资源。
5. 点击 `取消按钮` 触发 `popup_confirm`，关闭当前弹窗。

## 返回规则

- 弹窗可中断当前模态屏。
- 关闭后按记录恢复：优先恢复被中断模态；无法恢复则回 `基础屏`。

## 语义约束

- `取消按钮` 在本屏是“确认/继续”，不是“取消选择”。
- `关闭` 按钮仅用于黑市，不参与机会卡屏。
- `建筑升级_确定按钮` 仅用于建筑升级屏，不参与机会卡屏。

## 触发场景（保持业务不变）

包含但不限于：

- 机会卡抽取提示
- 购买失败 / 升级失败提示
- 偷窃成功/失败提示
- 黑市购买失败提示

以上业务来源不变，只变更 UI 承载画布。
