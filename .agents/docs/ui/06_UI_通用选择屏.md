# UI：通用选择屏（多选项弹窗）

通用选择屏用于展示标题 + 正文 + 若干选项按钮 + 可选的取消按钮。所有需要玩家从多个选项中选择的场景（地块卡牌、道具使用、黑市确认等）均复用此画布。

## Root

- `通用选择屏`（`ECanvas`，UIManagerNodes 注册名）

## 运行时节点映射（UIView.lua `ui_view.build_ui_state`）

- `通用选择_标题`（`ELabel`）：标题文字
- `通用选择_正文`（`ELabel`）：正文文字（由 `body_lines` 用 `\n` 拼接而成）
- `通用选择_选项_01` ~ `通用选择_选项_06`（`EButton`）：最多 6 个选项按钮，多余的隐藏
- `通用选择_取消`（`EButton`）：取消按钮，`allow_cancel=false` 时隐藏
- `通用选择_面板`（`EImage`）、`通用选择_选项区`（`EImage`）：容器节点

## 显隐与画布切换

打开选择屏（`_open_generic_choice`）时切换到通用选择屏画布，若黑市正在显示则先关闭。根据 `choice.options` 数量显示或隐藏对应的选项按钮，并设置按钮文字和可点击状态。关闭选择屏（`ui_view.close_choice_modal`）时隐藏画布，若有弹窗屏活跃则恢复弹窗屏，否则回到基础屏。

## 选项数据来源

所有选项由 `IntentDispatcher.open_choice` 统一注入 `game.turn.pending_choice`，再由 `UIChoice.build_choice_view` 构建为 `{ title, body, options, cancel_label, allow_cancel }` 结构。`kind` 字段标识场景类型，用于 `ChoiceResolver` 分发响应。

## 全部运行时显示场景（按 kind 分类）

### `item_phase_choice` — 道具使用阶段（`src/game/item/ItemPhase.lua`）

行动前 / 投骰后 / 行动后三个阶段均可触发。

| 字段 | 值 |
|------|----|
| title | `"行动前：使用道具？"` / `"投骰后：使用道具？"` / `"行动后：使用道具？"` |
| body | 当前阶段可用道具名称及用途，每行一个 |
| options | 可用道具 + `"丢弃道具"` 选项 |
| allow_cancel | `true` |
| cancel_label | `"结束阶段"` |

### `rent_card_prompt` — 强征卡 / 免费卡提示（`src/game/land/LandChoiceSpecs.lua`）

踩到他人地块且持有强征卡或免费卡时弹出。

| 字段 | 强征卡 | 免费卡 |
|------|--------|--------|
| title | `"是否使用强征卡"` | `"是否使用免费卡"` |
| body | `"支付 X 强制购入 地块名"` | `"免除本次租金"` |
| options | `使用` / `放弃` | `使用` / `放弃` |
| allow_cancel | `false` | `false` |

### `tax_card_prompt` — 免税卡提示（`src/game/land/LandChoiceSpecs.lua`）

踩到税务局且持有免税卡时弹出。

| allow_cancel | `false` |

### `steal_prompt` — 偷窃卡使用提示（`src/game/item/ItemSteal.lua`）

### `market_buy` — 黑市购买（`src/game/market/Market.lua`）

| 字段 | 值 |
|------|----|
| title | `"黑市"` |
| body | 可购买商品列表，每行 `商品名 - 价格 货币` |
| options | 各商品 |
| allow_cancel | `true` |
| cancel_label | `"不买"` |

### `landing_optional_effect` — 着陆可选效果（`src/game/effect/EffectPipeline.lua`）

| 字段 | 值 |
|------|----|
| title | 由调用方 `opts.optional_title` 指定 |
| body | 各效果的 label |
| options | 各效果 |
| allow_cancel | 由调用方 `opts.optional_allow_cancel` 指定 |
| cancel_label | 由调用方 `opts.optional_cancel_label` 指定 |
