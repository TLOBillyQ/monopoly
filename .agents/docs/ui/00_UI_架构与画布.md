# UI 架构与画布

## 真相源

- UI 节点与画布以 `Data/UIManagerNodes.lua` 为准。
- 本目录文档描述已落地的最终态，不包含迁移历史。

## 画布职责

| 画布 | 职责 |
|---|---|
| `基础屏` | 常驻 HUD、玩家信息、行动与道具入口 |
| `玩家选择屏` | 玩家目标类选择 |
| `位置选择屏` | 位置目标类与通用选择 |
| `遥控骰子屏` | 遥控骰子点数选择 |
| `建筑升级屏` | 地产可选操作确认 |
| `卡牌展示屏` | 全部弹窗展示 |
| `黑市屏` | 黑市购买流程 |
| `加载屏` | 初始化加载提示 |
| `调试屏` | 调试日志叠加层 |
| `破产展示屏` | 保留画布（不参与本轮选择系统） |

## 互斥与叠加

- `基础屏` 始终显示。
- 以下模态屏同一时刻最多显示一个：
  - `玩家选择屏`
  - `位置选择屏`
  - `遥控骰子屏`
  - `建筑升级屏`
  - `卡牌展示屏`
  - `黑市屏`
- `调试屏`独立叠加，不参与互斥。
- `加载屏`只在初始化阶段短时覆盖。

## 选择路由

| 条件 | 目标画布 |
|---|---|
| `choice.kind == "market_buy"` | `黑市屏` |
| `choice.kind == "remote_dice_value"` | `遥控骰子屏` |
| `choice.kind == "item_target_player"` | `玩家选择屏` |
| `choice.kind in {"roadblock_target", "demolish_target"}` | `位置选择屏` |
| `choice.kind in {"landing_optional_effect", "land_optional_effect"}` 且 `option_id` 全属于 `{buy_land, upgrade_land}` | `建筑升级屏` |
| 其他 kind | `位置选择屏` |

## 按钮语义

| 场景 | 提交 | 取消 |
|---|---|---|
| `卡牌展示屏` | `取消按钮`（语义=确认） | 无 |
| `玩家选择屏` | `玩家选择_槽位1..3`（点选即提交） | `取消按钮` |
| `位置选择屏` | `位置前/后/脚下`（点选即提交） | `取消按钮` |
| `遥控骰子屏` | `遥控骰子_选项_01..06` 直接提交 | `遥控骰子_取消` |
| `建筑升级屏` | `建筑升级_确定按钮` | `建筑升级_取消` |
| `黑市屏` | `黑市购买按钮` | `关闭` |

## 棋盘位移动画（2026-02-11）

- 玩家走格动画与 `move_effect` 强制位移动画统一走 `src/ui/PlayerMoveBehavior.lua`。
- 移动行为树固定三阶段：`PrepareMove -> RunSteps -> FinalizeMove`。
- 运行时默认开启行为树位移：`Config/RuntimeConstants.lua` 的 `player_move_bt_enabled = true`。
- 行为树移动期间会临时关闭“玩家 vs 玩家”碰撞，使用 pair 引用计数，最后一个会话结束后恢复。
- 玩家对齐高度统一走 `src/ui/PlayerPositioning.lua`：最低 `y = ground.y + player_ground_clearance`（默认 `1.5`）。
- 默认只在 `PrepareMove` 与 `FinalizeMove` 执行硬停+对齐；如需每步硬停可开启 `player_move_hard_stop_each_step = true`。
- UI 位置同步抑制规则：
  - `wait_move_anim` 且存在 `move_anim` 时抑制；
  - `wait_action_anim` 且 `action_anim.kind == "move_effect"` 时同样抑制。
