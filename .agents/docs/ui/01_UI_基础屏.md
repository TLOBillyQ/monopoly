# UI：基础屏

画布：`基础屏`

## 节点范围

- 操作节点：`行动按钮`、`托管按钮`、`托管_文本`
- 槽位节点：`道具槽位1`~`道具槽位5`
- 标签节点：`倒计时`
- 玩家信息节点：`玩家{1..4}头像/名字/现金/地块数量/总资产`

## 行为规则

1. `基础屏`常驻，不参与模态互斥。
2. `input_blocked=true` 或当前 role 不可操作时，以下节点隐藏或禁用：
   - `行动按钮`
   - `道具槽位1`~`道具槽位5`
   - `倒计时`
3. `托管按钮`、`托管_文本`保持可见；仅 `托管按钮`可点击，`托管_文本`只负责展示“自动：开/关”。
4. 道具槽位仅在 `item_phase_choice` 且 `item_choice_owner_id == display_player_id` 时可点击。
5. 玩家姓名优先使用 `GameAPI.get_role(player.id)` 对应 `Role.get_name()`；取不到时回退本地玩家名。
6. 玩家头像优先使用 `Role.get_head_icon()`；取不到时回退 `RuntimeRefs["空"]`。
7. 玩家出局后，姓名展示为 `昵称 (出局)`。

## 协作约束

- 模态屏打开时，基础屏不下线，但交互权限由输入锁策略统一处理。
- 模态关闭后，下一次 `refresh_panel` 负责按角色重新恢复可见与可交互状态。
