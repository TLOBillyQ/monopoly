# UI：基础屏

画布名：`基础屏`。始终作为底层显示，其他画布叠加其上。

## 节点清单

### 回合信息

| 节点名 | 类型 | 用途 |
|--------|------|------|
| `倒计时` | ELabel | 回合与倒计时 |

文本由 `UIPanel.build_turn_label` 生成：

- 倒计时激活时：`"回合: {turn_count} | 倒计时: {seconds}"`
- 无倒计时：`"回合: {turn_count}"`

### 玩家信息（1~4）

每个玩家占一组节点，`{i}` 为 1~4：

| 节点名 | 类型 | 用途 |
|--------|------|------|
| `玩家{i}名字` | ELabel | 玩家名称 |
| `玩家{i}现金` | ELabel | 现金数值 |
| `玩家{i}地块数量` | ELabel | 持有地块数 |
| `玩家{i}总资产` | ELabel | 现金 + 地块投资总额 |
| `玩家{i}头像` | EImage | 头像 |
| `玩家{i}信息` | EImage | 信息区背景 |
| `玩家{i}底板` | EImage | 面板底板 |
| `玩家{i}底板颜色` | EImage | 颜色标识 |

文本由 `UIPanel.build_player_statuses` 生成：

- 名字：`"{name} $cash"` 或 `"{name} (出局)"`
- 现金：`"现金: {cash}"`
- 地块数量：`"地块: {count}"`
- 总资产：`"总资产: {total}"`（总资产 = 现金 + 所有持有地块的已投资金额）

### 操作按钮

| 节点名 | 类型 | 用途 |
|--------|------|------|
| `行动按钮` | EButton | 推进回合，文字固定 `"下一回合"` |
| `托管按钮` | EButton | 切换自动模式 |
| `自动控制按钮` | ELabel | 显示自动状态（与托管按钮文字相同） |

托管文字由 `UIPanel.build_auto_label` 生成：`"自动：开"` / `"自动：关"`。

### 道具槽位

5 个道具槽位：`道具槽位1` ~ `道具槽位5`，类型均为 EImage。

- 通过 `image_texture` 设置道具图标，空槽显示 `空` 资源。
- 仅在 `item_phase_choice`（道具使用阶段）激活时可点击。
- 点击道具槽位等同于在选择屏中选择对应道具。

### 其他

| 节点名 | 类型 | 用途 |
|--------|------|------|
| `背景矩形` | EImage | 背景 |
| `遮挡` | EImage | 遮挡层 |
| `选项矩形` | EImage | 容器 |
| `取消按钮` | EButton | 已注册但当前未绑定行为 |

## 刷新流程

`ui_view.refresh_panel(state, ui_model)` 刷新全部基础屏节点：

1. 更新回合标签。
2. 遍历 4 个玩家行，设置名字、现金、地块数量、总资产。
3. 调用 `refresh_item_slots` 更新道具槽位图标和可点击状态。
4. 设置行动按钮文字 `"下一回合"`，设置托管按钮文字。

## 道具槽位刷新

`ui_view.refresh_item_slots(state, ui_model)`：

1. 从 `ui_model.item_slots` 取当前玩家背包物品 id 列表。
2. 检查 `ui_model.choice.kind == "item_phase_choice"` 决定是否可点击。
3. 有道具时设置对应图标；无道具时设置空图标并禁用点击。
