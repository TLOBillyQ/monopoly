# UI：基础屏（V2）

## 当前状态与目标状态

- 当前状态：基础屏逻辑可用，但与新多分屏路由的关系描述不足。
- 目标状态：基础屏作为常驻底层，负责 HUD 和可操作区，输入权限由回合与锁定策略统一控制。

## 画布与节点

画布：`基础屏`

主要交互节点：

- `行动按钮`
- `托管按钮`
- `自动控制按钮`
- `道具槽位1` ~ `道具槽位5`
- `倒计时`

玩家信息节点（示例）：

- `玩家1名字`、`玩家1现金`、`玩家1地块数量`、`玩家1总资产`
- `玩家2...`、`玩家3...`、`玩家4...`

## 行为规范

### 1) 常驻与隐藏

- `基础屏` 始终可见，不参与模态互斥。
- 当 `input_blocked=true` 或 role 不可操作时，隐藏/禁用以下分组：
  - `行动按钮`
  - `道具槽位1~5`
  - `倒计时`

### 2) 例外节点

以下节点不进入基础隐藏组：

- `托管按钮`
- `自动控制按钮`

说明：可见不等于可点击。观战或无映射 role 下仍应禁用交互。

### 3) 道具槽位交互

- 槽位显示按 `display_player_id` 读取背包映射。
- 仅在 `item_phase_choice` 且 `item_choice_owner_id == display_player_id` 时允许点击。
- 非操作方或锁定态，槽位必须禁用。

## 与其他画布协作

- 打开任意模态屏时，基础屏不隐藏，但交互权限可被锁定策略覆盖。
- 关闭模态屏后，不做“强制全开”，由下一次 `refresh_panel` 按角色权限恢复。

## 实现差距提醒

截至当前代码，基础屏本身可继续复用；后续实现重点在于把选择/弹窗从旧节点迁到新分屏，不在基础屏做额外分裂。
