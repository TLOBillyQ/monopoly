# UI：基础屏

画布名：`基础屏`。始终作为底层显示，其他画布叠加其上。

## 渲染范围规则

- 玩家信息区（`玩家1~4` 名字/现金/地块/总资产）继续全局一致渲染。
- 非玩家区按 role 渲染，role 映射规则是 `role.get_roleid() -> player.id`。
- role 映射失败（观战/异常）时，显示回退到当前回合玩家，但交互按观战处理（禁用）。

## 基础屏隐藏策略

基础屏新增“非玩家区隐藏组”：

- `base_hidden_nodes`：`行动按钮` + `道具槽位1~5`
- `base_hidden_labels`：`倒计时`

触发条件：

- 当前 role 不可操作（不是当前回合玩家）时隐藏；
- `input_blocked=true` 时隐藏。

例外（永不隐藏）：

- `托管按钮`
- `自动控制按钮`

## 操作按钮语义

| 节点名 | 类型 | 语义 |
|--------|------|------|
| `行动按钮` | EButton | 仅当前回合玩家可操作，非当前回合或输入锁定时隐藏/禁用 |
| `托管按钮` | EButton | 与当前回合无关。点击只切换“点击者对应玩家”的 `player.auto` |
| `自动控制按钮` | ELabel | 显示该 role 对应玩家的托管状态文案 |

托管文案由 `UIPanel.build_auto_label(player.auto)` 生成，为 `"自动：开"` / `"自动：关"`。

观战/未映射 role：

- 托管控件可见但禁用；
- 点击不会改变任何玩家状态。

## 道具槽位

5 个槽位：`道具槽位1`~`道具槽位5`。

- 槽位数据仍按 `display_player_id` 取 `item_slots_by_player`，并写入 `item_slot_item_ids_by_role[role_id]`。
- 仅在 `item_phase_choice` 且满足 `item_choice_owner_id` 时允许点击。
- 非当前回合或输入锁定时，道具槽位进入隐藏组（与行动按钮一致）。

## 刷新流程

`ui_view.refresh_panel(state, ui_model)`：

1. 全局刷新玩家信息。
2. 逐 role 计算渲染上下文（`display_player_id/can_operate`）。
3. 先应用基础屏隐藏组（节点+label）。
4. 再单独渲染托管控件（可见，不受隐藏组影响）。
5. 刷新道具槽位并记录 `item_slot_item_ids_by_role` 映射。

`ui_view.apply_input_lock(state)`：

- 锁定时继续禁用行动按钮、道具槽位、选择/黑市/弹窗按钮；
- 不禁用托管按钮与自动控制按钮；
- 不做“解锁后全量启用”，恢复由 `refresh_panel` 的逐 role 判定控制。
