# UI：基础屏

画布名：`基础屏`。始终作为底层显示，其他画布叠加其上。

## 渲染范围规则

- 玩家信息区（`玩家1~4` 名字/现金/地块/总资产）是**全局一致渲染**。
- 非玩家信息区（倒计时、行动按钮、托管按钮、自动控制按钮、道具槽位）按**客户端 role 单独渲染**。
- role 映射规则：`role.get_roleid() -> player.id`。
- 未映射 role（如观战）跟随当前回合玩家显示非玩家信息，但不允许操作。

## 节点清单

### 回合信息

| 节点名 | 类型 | 用途 |
|--------|------|------|
| `倒计时` | ELabel | 回合与倒计时 |

文本由 `UIPanel.build_turn_label` 生成：

- 倒计时激活时：`"回合: {turn_count} | 倒计时: {seconds}"`
- 无倒计时：`"回合: {turn_count}"`

### 玩家信息（1~4）

每个玩家占一组节点，`{i}` 为 1~4：

| 节点名 | 类型 | 用途 |
|--------|------|------|
| `玩家{i}名字` | ELabel | 玩家名称 |
| `玩家{i}现金` | ELabel | 现金数值 |
| `玩家{i}地块数量` | ELabel | 持有地块数 |
| `玩家{i}总资产` | ELabel | 现金 + 地块投资总额 |
| `玩家{i}头像` | EImage | 头像 |
| `玩家{i}信息` | EImage | 信息区背景 |
| `玩家{i}底板` | EImage | 面板底板 |
| `玩家{i}底板颜色` | EImage | 颜色标识 |

文本由 `UIPanel.build_player_statuses` 生成：

- 名字：`"{name} $cash"` 或 `"{name} (出局)"`
- 现金：`"现金: {cash}"`
- 地块数量：`"地块: {count}"`
- 总资产：`"总资产: {total}"`（总资产 = 现金 + 所有持有地块的已投资金额）

### 操作按钮

| 节点名 | 类型 | 用途 |
|--------|------|------|
| `行动按钮` | EButton | 推进回合，文字固定 `"下一回合"` |
| `托管按钮` | EButton | 切换自动模式 |
| `自动控制按钮` | ELabel | 显示自动状态（与托管按钮文字相同） |

托管文字由 `UIPanel.build_auto_label` 生成：`"自动：开"` / `"自动：关"`。

交互权限：

- 仅当前回合玩家（`role_id == current_player_id`）可点击 `行动按钮`、`托管按钮`。
- 非当前回合玩家与观战端显示同样按钮文本，但触摸禁用。
- 事件分发层会再按 `actor_role_id` 做越权拦截，不只依赖 UI 禁用。
- 当当前回合玩家为 AI 且未开启托管时，系统按 `GameplayRules.ai_auto_turn_interval_seconds` 自动推进回合，无需点击按钮。

### 道具槽位

5 个道具槽位：`道具槽位1` ~ `道具槽位5`，类型均为 EImage。

- 显示当前客户端对应玩家的背包内容（不是“当前回合玩家”的通用内容）。
- 通过 `image_texture` 设置道具图标，空槽显示 `空` 资源。
- 仅在 `item_phase_choice`（道具使用阶段）且该 role 有操作权时可点击。
- 点击道具槽位等同于在选择屏中选择对应道具。

观战/未映射 role：

- 道具槽位显示跟随当前回合玩家。
- 始终不可点击。

### 其他

| 节点名 | 类型 | 用途 |
|--------|------|------|
| `背景矩形` | EImage | 背景 |
| `遮挡` | EImage | 遮挡层 |
| `选项矩形` | EImage | 容器 |
| `取消按钮` | EButton | 已注册但当前未绑定行为 |

## 刷新流程

`ui_view.refresh_panel(state, ui_model)` 刷新全部基础屏节点：

1. 全局渲染玩家信息（4 个玩家行）。
2. 遍历所有 role，逐 role 渲染倒计时、按钮、道具槽位。
3. 依据 role 是否映射到当前回合玩家，设置按钮与道具槽位点击权限。
4. 写入 `item_slot_item_ids_by_role`，供事件路由按点击者取槽位道具。

## 道具槽位刷新

`ui_view.refresh_item_slots(state, ui_model)`：

1. 先按 `display_player_id` 从 `ui_model.item_slots_by_player` 取道具列表。
2. 再按 `allow_interact` + `item_phase_choice` + `item_choice_owner_id` 判定可点击。
3. 有道具时设置对应图标；无道具时设置空图标并禁用点击。
4. 把当前 role 的槽位映射写入 `state.ui.item_slot_item_ids_by_role[role_id]`。
