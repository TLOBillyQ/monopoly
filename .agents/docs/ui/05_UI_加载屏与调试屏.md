# UI：加载屏与调试屏（V2）

## 当前状态与目标状态

- 当前状态：两者已有节点与基本功能。
- 目标状态：保持轻量，不卷入选择系统与模态路由。

## 加载屏

画布：`加载屏`

节点：

- `加载提示`（ELabel）

规则：

1. 初始化阶段显示加载屏。
2. 初始化完成后隐藏加载屏，回到 `基础屏`。
3. 加载屏不参与正常回合内的选择互斥。

## 调试屏

画布：`调试屏`

节点：

- `日志`（ELabel）

规则：

1. 调试屏是叠加层，不参与模态互斥。
2. 通过 `set_debug_visible` 控制显隐。
3. 通过 `set_debug_log` 更新日志文本。
4. 切换其他画布时，若调试开启应保持可见。

## 约束

- 调试屏仅用于观测，不承载业务交互按钮。
- 加载屏仅用于启动过程，不承担弹窗/选择功能。
