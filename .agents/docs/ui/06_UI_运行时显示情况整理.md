# UI 运行时显示情况整理：弹窗屏与通用选择屏

本文档整理弹窗屏（`弹窗屏`/`Canvas_Popup`）和通用选择屏（`通用选择屏`/`Canvas_Choice`）在运行时的所有显示场景、触发时机、显示内容和交互配置。

## 1. 总览

### 1.1 UI 画布类型

项目使用 Canvas 切换机制管理 UI 显示：

- **基础屏** (`CANVAS_BASE` = `"基础屏"`)：主游戏界面（HUD、地图、玩家信息等）
- **弹窗屏** (`CANVAS_POPUP` = `"弹窗屏"`)：模态弹窗，用于提示、确认操作
- **通用选择屏** (`CANVAS_CHOICE` = `"通用选择屏"`)：多项选择界面
- **黑市屏** (`CANVAS_MARKET` = `"黑市屏"`)：黑市购买界面
- **调试屏** (`CANVAS_DEBUG` = `"调试屏"`)：调试信息显示

### 1.2 UI 切换规则

**互斥关系**：
- 同一时刻只有一个主画布可见（基础屏、弹窗屏、选择屏、黑市屏之一）
- 调试屏可与其他画布共存（若 `debug_visible = true`）

**弹窗屏与选择屏的层级关系**：
- 弹窗屏可叠加在选择屏或黑市屏之上
- 关闭弹窗时，会返回之前的画布（保存在 `popup_return_canvas` 中）
- 弹窗优先级最高：当弹窗激活时，选择屏和黑市屏的交互被阻断

**UI 状态管理**：
```lua
-- 代码位置：src/ui/UIView.lua:85-129
ui_state = {
  choice_active = false,       -- 通用选择屏是否激活
  market_active = false,       -- 黑市屏是否激活
  popup_active = false,        -- 弹窗屏是否激活
  popup_return_canvas = nil,   -- 弹窗关闭后返回的画布
  pending_choice_id = nil,     -- 当前等待的选择 ID
}
```

### 2.3 弹窗触发流程

```

  ↓

  ↓
IntentDispatcher.push_popup (src/game/intent/IntentDispatcher.lua:34-43)
  ↓
ui_port:push_popup(payload)
  ↓
UIView.push_popup (src/ui/UIView.lua:384-403)
  ↓
切换到 CANVAS_POPUP，设置标题、正文、按钮文本

```

### 2.4 弹窗关闭流程

```
用户点击确认按钮
  ↓

  ↓

  ↓
切换回 popup_return_canvas 指定的画布

  - 若之前在选择屏：返回 CANVAS_CHOICE
  - 否则：返回 CANVAS_BASE
```

**显示配置**：

## 4. 特殊场景：黑市屏

虽然黑市屏不是通用选择屏，但它与选择系统紧密相关，值得特别说明。

### 4.1 黑市屏触发条件

**触发场景**：当 `choice.kind == "market_buy"` 且黑市 UI 组件已加载  
**代码位置**：`src/ui/UIView.lua:359-363`  
```lua
if choice.kind == "market_buy" and market_ui.is_panel_ready() then
  _open_market_panel(state, choice, choice_id, market)
else
  _open_generic_choice(state, choice, choice_id)
end
```

### 4.2 黑市屏与通用选择屏的选择

- 若黑市 UI 组件就绪（`market_ui.is_panel_ready() == true`），使用黑市屏
- 否则，回退到通用选择屏显示黑市购买界面

## 5. UI 阻塞与输入控制

### 5.1 输入阻塞机制

**代码位置**：`src/ui/UIView.lua:210-259`  
**阻塞条件**：`ui.input_blocked == true`

**被阻塞的交互元素**：
- 行动按钮、托管按钮、自动控制按钮
- 道具槽位
- 通用选择屏选项按钮和取消按钮
- 黑市屏商品按钮、确认和取消按钮
- 弹窗屏确认按钮

### 5.2 弹窗优先级

当弹窗激活时：
- 弹窗之外的所有交互被禁用
- 只有弹窗确认按钮可交互（如果 `input_blocked == false`）
- 关闭弹窗后，恢复之前画布的交互状态

## 7. 实现要点与注意事项

### 7.1 选择 ID 与去重

每个选择都有唯一的 `choice.id`（递增序列号），防止重复显示同一个选择：
```lua
-- 代码位置：src/ui/UIView.lua:353-356
if state.pending_choice_id == choice_id
    and (state.ui.choice_active or state.ui.market_active) then
  return  -- 已经显示，不重复打开
end
```

### 7.2 阶段标题前缀

选择屏标题可能带有阶段前缀（如 `[行动前]`），由 `_build_phase_title` 函数生成：
```lua
-- 代码位置：src/ui/UIChoice.lua:33-42
local phase_name = game.turn.item_phase_active
if phase_name then
  local label = _build_phase_label(phase_name)  -- "行动前"/"投骰后"/"行动后"
  return "[" .. label .. "] " .. base_title
end
```

### 7.3 超时与自动处理

选择有超时机制，超时后自动选择默认选项（托管玩家可提前触发）：
```lua
-- 代码位置：src/game/turn/TickTimeout.lua:93-122
state.pending_choice_elapsed = state.pending_choice_elapsed + dt
if state.pending_choice_elapsed >= timeout then
  -- 超时，自动选择
end
```

## 10. 总结

本文档整理了弹窗屏和通用选择屏在运行时的所有显示情况，包括：

**弹窗屏**：
- 7 种触发场景（偷窃失败/成功、背包满、黑市无商品、购买/升级失败、黑市购买失败）
- 简单的确认交互（仅一个按钮）
- 可叠加在其他画布之上

**通用选择屏**：
- 10 种 Kind 类型（道具使用、租金卡、免税卡、偷窃卡、偷道具、丢弃道具、拆迁目标、黑市购买、座驾更换、地块可选效果）
- 最多 6 个选项
- 可选的取消按钮
- 可带阶段前缀的标题

**层级关系**：
- 弹窗优先级最高
- 选择屏和黑市屏互斥
- 基础屏为默认画布

此文档可用于：
- 策划理解所有 UI 显示场景
- 美术制作对应的 UI 资源
- 程序员维护和扩展 UI 系统
- QA 测试人员验证 UI 显示逻辑
