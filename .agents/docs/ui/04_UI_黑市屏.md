# UI：黑市屏（V2）

## 当前状态与目标状态

- 当前状态：黑市屏逻辑已存在，但文档仍引用旧“通用选择屏/弹窗屏”叙述。
- 目标状态：黑市屏继续承接 `market_buy`，并与新选择系统、机会卡屏语义解耦。

## 画布与节点

画布：`黑市屏`

### 控制按钮

| 节点名 | 类型 | 语义 |
|---|---|---|
| `黑市购买按钮` | EButton | 确认购买 |
| `关闭` | EButton | 取消/关闭黑市 |

### 预览与价格

| 节点名 | 类型 | 语义 |
|---|---|---|
| `售价：100` | ELabel | 显示 `售价：{price} {currency}` |
| `选中卡牌` | EImage | 当前选中商品预览 |

### 商品槽位（1~10）

- `黑市购买项1..10`
- `道具名称1..10`
- `底框1..10`

## 打开与刷新规则

`market_buy` 路由到黑市屏后：

1. 切换到 `黑市屏`。
2. 刷新最多 10 个商品槽位，按 options 原顺序映射。
3. 自动选中首个可选商品并更新价格/预览。
4. `关闭` 是否可见遵循 `allow_cancel`。

## 交互规则

- 点击 `黑市购买项{i}`：仅更新选中项（`market_select`）。
- 点击 `黑市购买按钮`：提交当前选中项（`market_confirm -> choice_select`）。
- 点击 `关闭`：发送 `choice_cancel`。

## 与其他画布关系

- 黑市与玩家/位置/遥控骰子/建筑升级/机会卡均互斥。
- 若黑市操作失败，需要弹提示时，提示应走 `机会卡屏`，不再依赖旧弹窗屏文档语义。

## 语义边界

- `关闭` 仅属于黑市，不作为通用取消按钮。
- `取消按钮` 不参与黑市流程。
