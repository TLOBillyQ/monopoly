# UI：黑市屏

画布名：`黑市屏`。专用于 `market_buy` 类型的选择，提供商品列表、价格显示、选中预览。当 `market_ui.is_panel_ready()` 返回 `true` 时使用本画布，否则降级到通用选择屏。

## 节点清单

节点名和布局定义在 `MarketLayout.lua`。

### 控制按钮

| 节点名 | 类型 | 用途 |
|--------|------|------|
| `黑市购买按钮` | EButton | 确认购买 |
| `关闭` | EButton | 关闭 / 取消 |

### 价格与预览

| 节点名 | 类型 | 用途 |
|--------|------|------|
| `售价：100` | ELabel | 价格标签，文字动态更新为 `"售价：{price} {currency}"` |
| `选中卡牌` | EImage | 选中商品的大图预览 |

### 商品列表（1~10）

每个商品占一组 3 个节点，`{i}` 为 1~10：

| 节点名 | 类型 | 用途 |
|--------|------|------|
| `黑市购买项{i}` | EImage | 商品按钮（可点击） |
| `道具名称{i}` | ELabel | 商品名称（不可点击） |
| `底框{i}` | EImage | 品质底框（不可点击） |

### 品质底框资源

| 等级 | 资源 key |
|------|---------|
| tier 1 | `"lv1"` |
| tier 2 | `"lv2"` |
| tier 3 | `"lv3"` |
| 空 | `"空"` |

等级由 `cfg.tier` 决定：`< 1` → 1，`>= 3` → 3，其余 → 2。

## 打开黑市

`_open_market_panel(state, choice, choice_id, market)` 在 `UIView.lua` 中：

1. 切换到黑市屏画布。
2. 若通用选择屏活跃则隐藏。
3. 调用 `market_view.refresh_market` 填充商品。

## 刷新商品

`market_view.refresh_market(state, market)` 在 `MarketView.lua` 中：

1. 显示黑市屏容器，置 `market_active = true`。
2. 遍历最多 10 个商品槽位：
   - 有商品：设置名称标签、显示按钮和底框、设置底框品质图片、禁用名称和底框的点击。
   - 无商品：隐藏按钮、名称、底框。
3. 禁用 `选中卡牌` 的点击。
4. 显示确认按钮；根据 `allow_cancel` 显示或隐藏关闭按钮。
5. 自动选中首个商品（调用 `select_market_option`）。

## 选中商品

`market_view.select_market_option(state, option_id)`：

1. 存储 `pending_choice_selected_option_id`。
2. 更新价格标签：`"售价：{price} {currency}"`。
3. 更新 `选中卡牌` 图片为对应商品图标。

## 关闭黑市

`market_view.close_market_panel(state)`：

1. 隐藏黑市屏容器。
2. 清空 `market_active`、`market_choice_option_ids`、`pending_choice_selected_option_id`。
3. 清空价格标签文字。
4. 禁用全部名称标签和底框的点击。
5. 将 `选中卡牌` 图片重置为空资源。

## 交互流程

1. 玩家点击 `黑市购买项{i}` → `market_select` intent → 更新选中状态和预览。
2. 玩家点击 `黑市购买按钮` → `market_confirm` intent → 转为 `choice_select` 分发。
3. 玩家点击 `关闭` → `choice_cancel` intent。

## 购买失败

购买操作在 `Market.lua` 中执行。失败时通过 `monopoly_event.market.buy_failed` 事件弹出弹窗（见 `02_UI_弹窗屏.md`），不关闭黑市面板。
