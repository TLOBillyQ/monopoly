# ReadingDiscipline

核心立场：实现者，不是读者。

## 1 按需检索，禁止通读

- API 文档是数据库，不是教材。不顺序阅读，不"先了解全貌"。
- 每次只查一个具体 symbol（函数、类型、枚举）。有明确问题才打开文档。

## 2 只取最小信息

- 允许读取：函数签名、单个类型定义、小段示例代码。
- 禁止读取：完整头文件、连续浏览相关 API、"以防万一"的预读。

## 3 不确定时假设，不阻塞

- 行为不明确 → 用最安全的假设继续实现。
- 用 TODO 标注假设点，不因"文档不清楚"停下来。

## 4 实现优先级

1. 仓库内已有的代码模式（最可靠）
2. 最小可运行实现
3. 保守默认值
4. TODO 标注待确认处

## 5 推进节奏

- 小步实现，频繁验证。
- 不问"下一步做什么"，自行判断。
- 卡住时选风险最低的路径继续。
