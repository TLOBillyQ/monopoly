# 蛋仔大富翁 - Lua游戏原型

## 功能实现

### 已实现的核心系统

1. **玩家系统** (player.lua)
   - 玩家创建和管理
   - 金币管理
   - 地块所有权
   - 道具卡槽（最多5个）
   - 附身神状态（天使、财神、穷神）
   - 座驾系统
   - 停留状态（医院、深山）
   - AI决策逻辑框架

2. **地块系统** (property.lua)
   - 地块类型：空地、起点、机会卡、道具卡、医院、深山、税务局、黑市
   - 建筑等级：空地、房屋、别墅、高楼
   - 购买、加盖、租金计算
   - 路障和地雷系统
   - 地块事件处理

3. **道具卡系统** (item.lua)
   - 19种道具卡
   - 使用时机：行动前、主动使用、被动触发
   - 权重随机抽取
   - 道具效果框架

4. **机会卡系统** (chance.lua)
   - 24张预设机会卡
   - 事件类型：获得/扣除金币、座驾、建筑、移动等
   - 事件目标：自己、全体、其他
   - 权重随机抽取
   - 天使附身保护

5. **游戏流程控制** (game.lua)
   - 回合制系统
   - 阶段管理：行动前、投骰子、移动、事件、行动后
   - 自动超时机制
   - 破产和胜利判定
   - AI和人类玩家支持

6. **配置系统** (config.lua)
   - 游戏参数配置
   - 地块配置
   - 道具配置
   - 颜色和窗口配置

## 游戏规则（根据策划案）

### 基本规则
- 支持2-4人对战，单人游戏自动加入3个AI
- 初始金币：100,000
- 胜利条件：
  1. 其他角色全部淘汰
  2. 游戏时间结束，总资产最高者获胜
- 淘汰条件：金币归零

### 回合流程
1. **行动前阶段**：可使用道具，确认后投骰子
2. **行动中阶段**：自动投骰子、移动、触发事件
3. **行动后阶段**：可使用道具，确认后结束回合

### 地块机制
- 购买价格：基础价格
- 加盖价格：基础价格 × (2^加盖次数)
  - 房屋 = 基础价格 × 2
  - 别墅 = 基础价格 × 4
  - 高楼 = 基础价格 × 8
- 租金：最后一次加盖费用 × 0.5
- 相邻地块同属一人时租金叠加

### 特殊地块
- **起点**：经过获得2000金币
- **黑市**：经过时可购买道具
- **医院**：支付1000金币，停留2回合
- **深山**：停留2回合，期间无法收租
- **税务局**：缴纳现金50%的税

## 代码结构说明

### 主要数据结构

**Player（玩家）**
```lua
{
    id,              -- 玩家ID
    name,            -- 玩家名称
    is_ai,            -- 是否AI
    money,           -- 金币
    properties,      -- 拥有的地块
    position,        -- 当前位置
    state,           -- 状态
    items,           -- 道具卡
    vehicle,         -- 座驾
    angel_turns,      -- 天使回合数
    ...
}
```

**Property（地块）**
```lua
{
    id,              -- 地块ID
    name,            -- 名称
    type,            -- 类型
    base_price,       -- 基础价格
    owner,           -- 所有者
    building_level,   -- 建筑等级
    has_roadblock,    -- 是否有路障
    has_landmine,     -- 是否有地雷
    ...
}
```

## 扩展开发

### 添加新地块
在 `config.lua` 中的 `Config.properties` 添加配置

### 添加新道具
1. 在 `item.lua` 中的 `Item.Type` 添加类型
2. 在 `Item.use()` 中实现效果
3. 在 `config.lua` 中添加配置

### 添加新机会卡
在 `chance.lua` 中的 `Chance.create_all_chances()` 添加卡片

### 自定义AI策略
修改 `player.lua` 中的 `Player.make_ai_decision()` 函数
