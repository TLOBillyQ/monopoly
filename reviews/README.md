## 评审标准

所有代码评审基于以下标准：

1. **AGENTS.md 编码规则** - 项目特定的代码规范
2. **SOLID 原则** - 面向对象设计原则
3. **清洁架构** - 依赖方向与层次分离
4. **代码简洁性** - "优先删除或复用代码"

## 如何使用评审报告

1. **开发者** - 在修改代码前查阅相关模块的评审意见
2. **Code Review** - 使用评审报告中的检查清单验证新代码
3. **重构计划** - 参考"修复路线图"制定技术债务清理计划
4. **新人培训** - 了解项目的代码质量标准和常见问题

## 评审周期

建议每个重大功能完成后或每月进行一次代码评审，保持代码质量持续改进。
