# 落地路线图（基于 reviews/gaps.md）

## 目标与原则
- 目标：补齐设计与代码差异，保持行为/输出/边界条件一致（仅补齐缺失）。
- 原则：最小改动、优先删除或复用代码、限制影响范围。

## 阶段 0：现状对齐与验证准备（1-2 天）
- 梳理每条 gap 的当前入口与调用链（见“分解清单”）。
- 补充可回归的验证脚本清单与手工用例（不改逻辑）。
- 明确设计文档中对每条 gap 的最终期望文本（必要时标注冲突点）。

交付物：
- gap 分解清单（文件/函数/触发路径）
- 回归检查清单（手动/自动）

## 阶段 1：规则与胜负条件（2-3 天）
优先处理全局规则与胜负逻辑，避免后续联动返工。
- 胜利条件：加入“时间到以资产最高获胜（同资产并列获胜）”逻辑。
- 行动超时：实现 10s 自动确认并接入 `action_timeout_seconds`。
- 破产触发：现金为 0 即淘汰（与现有负数触发合并）。
- 同格人数上限：每格最多 4 人限制与处理策略。

交付物：
- 规则类行为补齐的实现与用例

## 阶段 2：移动与地形交互（2-4 天）
修复移动过程中的时序与交互差异。
- 路障：停一整回合（含状态/回合跳过）并清除路障。
- 地雷触发时机：事件处理后再检查地雷。
- 偷盗卡：移动中触发并继续剩余步数（中断-恢复）。

交付物：
- 移动流程与落地流程的修正

## 阶段 3：卡牌与道具体验（3-5 天）
补齐卡牌与道具相关交互与策略。
- 机会卡数量：按设计 34 张处理（对 3035-3037 作出取舍）。
- 清障卡：机器人分叉清全路径。
- 物品丢弃：新增丢弃流程与 UI/交互入口。
- 物品满提示：提示用户而非仅日志。
- AI 道具使用：覆盖 mine 卡与“可用即用”策略。

交付物：
- 卡牌/道具功能补齐与用户提示完善

## 阶段 4：经济与结算一致性（2-3 天）
对“钱”的增减路径做一致性补齐。
- 市场强制移动：按设计进入市场 UI（人类玩家可选，AI 走策略）。
- 地产购买/升级：余额不足仍展示选项，确认后失败提示。
- 富神/穷神：统一对“加钱/扣钱/转账”效果的倍率适用。

交付物：
- 经济类规则与 UI 交互一致化

## 分解清单（按文件）
- `src/game.lua`：胜利条件；每格人数上限。
- `src/config/constants.lua`：超时配置接入。
- `src/gameplay/movement_service.lua`：路障停回合；偷盗卡中断/恢复。
- `src/config/landing_effects.lua`：地雷触发时序。
- `src/gameplay/item_post_effects.lua`：清障卡分叉；破产触发。
- `src/gameplay/chance.lua`：胜负/破产逻辑；富穷神倍率；偷盗卡时序。
- `src/gameplay/market_service.lua`：市场 UI 流程接入。
- `src/gameplay/land.lua`：不足资金时的购买/升级提示与失败处理。
- `src/gameplay/item_phase.lua`：物品丢弃流程入口。
- `src/gameplay/choice_handlers/item_choice_handler.lua`：丢弃选择处理。
- `src/gameplay/item_strategy.lua`：AI 道具使用覆盖。
- `src/gameplay/item_inventory.lua`：物品满提示。
- `src/config/chance_cards.lua`：机会卡数量对齐。
- `src/gameplay/land_actions.lua`：富穷神倍率影响范围。
- `src/gameplay/landing.lua`：移动中事件/偷盗卡触发点。

## 风险与决策点
- 机会卡数量调整：是否删除 3035-3037 或在策划案中补充说明。
- 破产边界：现金为 0 的淘汰时机与 UI 提示文案。
- 市场强制移动 UI：是否已有 UI 资源与交互协议。
- 分叉清障卡：路径判定与性能开销。

## 回归建议
- 关键链路：移动 -> 落地 -> 事件 -> 经济结算 -> 结束判定。
- 道具链路：抽卡 -> 入包 -> 使用 -> 丢弃 -> 满包提示。
- AI 链路：行动 -> 道具使用 -> 市场/购买决策。
