# ReadingDiscipline
## 核心规则（必须遵守）
### 1. 禁止通读文档
- 不得顺序阅读完整 API 文档或头文件；不得尝试“先理解整个 SDK”；大型 API 视为**外部数据库**而非上下文。
### 2. 仅允许问题驱动的 API 查询
- 仅在有明确实现问题且指向具体 symbol（函数/类型/枚举/宏）时查看；每次只针对一个问题或一个 symbol。
### 3. 不确定时优先假设
- 行为不明→最小安全假设→继续实现→必要时注释标注；禁止因“文档不清楚”阻塞进度。
## API 使用边界
- 允许：函数签名、单个类型定义、小范围代码片段。
- 禁止：完整头文件、连续探索多个相关 API、“以防万一”的文档阅读。
---
## 实现优先级
1. 仓库内已有代码模式
2. 最小可编译实现
3. 保守默认值
4. TODO 标注不确定点
---
## 推进规则
- 小步实现，频繁提交；不问“下一步做什么”；卡住选风险最低路径。
---
## 总原则
**Codex 是实现者，不是读者。** 不确定 → 假设 → 实现 → 前进。
