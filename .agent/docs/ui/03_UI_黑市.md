# UI：黑市（购买道具/座驾）

对应策划：经过黑市自动弹出；可购买道具与座驾；道具进卡槽；卡槽满提示无法购买；已有座驾时提示是否更换；关闭黑市后继续未走完步数。

## Root

- `Canvas_Shop`（`ECanvas`）

## 层级结构（命名 / 类型）

```text
Canvas_Shop (ECanvas)
└─ Img_Shop_Root (EImage)
   ├─ Img_Shop_Mask (EImage)                      # 压暗+阻断点击
   └─ Img_Shop_Panel (EImage)
      ├─ Lbl_Shop_Title (ELabel)                  # “黑市”
      ├─ Btn_Shop_Close (EButton)                 # “关闭”
      ├─ Img_Shop_Tabs (EImage)
      │  ├─ Btn_Shop_Tab_Items (EButton)          # “道具”
      │  └─ Btn_Shop_Tab_Vehicles (EButton)       # “座驾”
      ├─ Img_Shop_List (EImage)                   # 商品列表容器
      │  ├─ Img_Shop_Item_01 (EImage)             # 商品卡 01..N
      │  │  ├─ Img_Shop_Item_01_Icon (EImage)
      │  │  ├─ Lbl_Shop_Item_01_Name (ELabel)
      │  │  ├─ Lbl_Shop_Item_01_Price (ELabel)
      │  │  └─ Btn_Shop_Item_01_Buy (EButton)     # “购买”
      │  ├─ Img_Shop_Item_02 (EImage)
      │  └─ Img_Shop_Item_NN (EImage)
      └─ Img_Shop_Toast (EImage)
         └─ Lbl_Shop_ToastText (ELabel)           # “卡槽已满/无法购买”等
```

## 交互点

- `Btn_Shop_Close`
- `Btn_Shop_Tab_Items` / `Btn_Shop_Tab_Vehicles`
- `Btn_Shop_Item_XX_Buy`

## 规则落地（必须）

- 卡槽满：购买道具失败，显示 `Lbl_Shop_ToastText`（例如“你的卡槽满了”），不关闭商店
- 已有座驾：购买座驾前弹出确认（可复用 `Canvas_Popup` 的结构：提示“是否更换座驾”，按钮“购买/算了”）
