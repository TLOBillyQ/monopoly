## 主要优化策略

### 1. DRY 原则 (Don't Repeat Yourself)
- 提取重复的 `get_service` 到 `src/util/services.lua`
- 提取重复的 `tile_state` 到 `src/util/game_state.lua`
- 统一错误处理模式

### 2. 数据驱动设计
- 将命令式的 handler 函数转换为配置驱动
- 使用配置表代替大量相似函数
- 通用处理函数 + 配置数据

### 3. 抽象与封装
- 创建模态对话框构建器
- 提取公共渲染辅助函数
- 统一服务访问接口

### 4. 简化与合并
- 合并相似的目标查找函数
- 统一机会牌效果处理
- 减少条件分支嵌套

### 5. 配置优化
- 使用默认值减少冗余字段
- 更紧凑的配置格式
- 移除未使用的字段

## 风险管理

### 已识别风险
1. **功能破坏** - 通过小步重构和频繁测试缓解
2. **新 bug 引入** - 通过代码审查和测试覆盖缓解
3. **性能下降** - 通过性能测试和避免过度抽象缓解

### 回滚策略
- 使用 git 分支隔离每个重构阶段
- 保持小的、可回滚的提交
- 遇到问题立即停止并评估
