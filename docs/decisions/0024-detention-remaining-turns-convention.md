# ADR 0024 - 扣留剩余回合采用「含当前回合」口径并跨界面统一；路障不属于扣留

## 状态

Proposed（2026-06-30，等待用户 review）。

## 背景

被扣留玩家身上有两个显示「剩余回合」的界面——3D 状态标签与扣留提示 tips——目前口径不一致：最后一个冻结回合出现「3D 显剩余 1、tips 显剩余 0」的分歧。需要先把「扣留剩余回合」这个玩家可见数字定义清楚，两个界面才能统一。

## 决定

1. **口径 = 含当前回合（inclusive）。** 「扣留剩余回合」表示还要被冻结的回合数、含当前正在进行的冻结回合；被扣留期间恒不小于 1，计到 0 即表示扣留解除、不再显示。玩家永不看到「剩余 0 回合」。停留 2 回合时，落地与两个冻结回合依次显示 2、2、1。

2. **两界面显示同一个数字。** 3D 状态标签与扣留提示在每个冻结回合显示相同的「扣留剩余回合」，不得分歧。

3. **扣留只含医院、深山（各 2 回合）。** 路障是「强制停在该格、截断剩余步数」的触发型障碍，不带回合数、不冻结下一回合、不属于扣留。权威依据：设计源 `道具表.xlsx` 路障卡（id=2004）说明为「任何玩家经过此地时**强制停留**」（无回合数，与「住院/深山迷路惩罚」分列），以及 `features/game/turn_flow.feature`「路障停留不影响下一回合行动」。

4. **神灵状态沿用同口径。** 财神/穷神/天使的剩余回合本就采用「含当前回合」且归零即消失，本次行为不变，仅补一条回归护栏锁定该口径。

## 取舍

- 选用「含当前回合」而非「不含当前回合（减后值）」：后者会在最后一个冻结回合显示「剩余 0 回合」（仍不能动却显示 0），语义矛盾；前者与落地「需停留 2 回合」宣告自然倒数对齐，且「0 = 自由 = 不显示」天然成立。
- 路障归类有一处过时文档冲突：`docs/product/map.md` 仍写「路障…被扣留 1 回合（`stay_turns = 1`）」，与代码、验收、设计源均矛盾。本 ADR 不改该产品文档，仅记录该冲突待产品文档负责人订正。
