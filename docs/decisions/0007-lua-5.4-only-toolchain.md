# ADR 0007 — 单一 Lua 5.4 工具链，退场 5.5 链路

**Status**: Stable (2026-05-22, Phase 1-7 全部执行完毕；8 项验收全绿)
**Trigger**: 2026-05-22 本机环境盘点发现 lua 5.5 与项目目标 5.4 半迁移共存
**Related**: `CLAUDE.md`（项目锁 Lua 5.4）、`tools/quality/verify_full.lua`、`tools/quality/coverage.lua`、`.luacheckrc`、`.luarc.json`、`docs/guides/migrate-lua-5.4-only.md`（执行 playbook）

---

## 上下文（Why）

项目所有静态约束都指向 Lua 5.4：

| 来源 | 声明 |
|---|---|
| `CLAUDE.md` | "Lua 5.4" |
| `.luarc.json` | `runtime.version = "Lua 5.4"` |
| `.luacheckrc` | `std = "lua54"` |
| `tools/quality/verify_full.lua:18-23` | 显式查找 `lua5.4` 二进制 |
| `tools/quality/coverage.lua:64` | 读取 `LUA54_LUA_PATH`、`LUA54_LUA_CPATH` |

但 2026-05-22 本机实测呈现**矛盾的半迁移态**：

| 通道 | 实际跑在 |
|---|---|
| `lua` 命令（brew 的 `lua` 5.5 已 linked） | **Lua 5.5.0** |
| `verify_full.lua` 的 lint 步骤（走 `lua5.4` 绝对路径） | Lua 5.4.8 |
| `verify_full.lua` 的 contract / guards 步骤（裸 `busted --run`） | **Lua 5.5**（`/opt/homebrew/bin/busted` shim 硬编码 `#!/opt/homebrew/opt/lua/bin/lua5.5`） |
| `coverage.lua`（用 `luarocks --lua-version=5.4 list` 解析 busted） | Lua 5.4.8 |

5.5 是 5.4 的语法 + stdlib 超集，所以测试碰巧能跑——但项目"在 5.4 上验证"的承诺被 brew shim 悄悄打破。`.luacheckrc` 的注释也承认这点："luacheck 1.2.0 不识别 lua55；等 luacheck 升级后再切。"

同时本机有 12/13 个 rocks 已经装在 `~/.luarocks/lib/luarocks/rocks-5.4/` 用户树下，证明此前已尝试过 5.4 迁移但未完成。

继续维持半迁移态的代价是隐性的——任何 Lua 5.5-only 语法（例如 5.5 引入的整数操作差异）一旦溜进测试代码，本地通过、其他人机器或 CI 上跑 5.4 时炸。

---

## 决策（What）

### D1 — 唯一 Lua 来源：`lua@5.4` (brew keg) brew-link 为 `lua`

`/opt/homebrew/opt/lua@5.4` 是系统上唯一的 lua 二进制 + dylib 来源。通过 `brew link lua@5.4` 让 `/opt/homebrew/bin/lua` 指向它。

**理由**：brew 的 keg-only 设计就是为多版本共存准备的，`lua@5.4` formula 已存在，无需自建源。link 后 `lua --version` 与 `lua5.4 --version` 一致，消除"裸 `lua` 命令跑哪个版本"的歧义。

### D2 — 退场 brew 的 lua 5.5 完整链路

卸载 `lua`（5.5）、`luarocks`（绑 5.5）、`gnuplot`（依赖 5.5）三个 brew formula。

**理由**：
- brew `luarocks` 3.13.0 的 formula 硬依赖 `lua`（5.5），shebang 是 `#!/opt/homebrew/opt/lua/bin/lua5.5`。无 `luarocks@5.4` formula，留着只会持续往 5.5 树装 rocks。
- 由 `luarocks` 安装到 `/opt/homebrew/bin/` 下的 shim（如 `busted`）同样硬编码 5.5 路径，是 contract / guards 错跑 5.5 的根因。
- `gnuplot` 是唯一非 luarocks 的 lua 5.5 反向依赖。本项目不用它，留着就是阻塞 `brew uninstall lua` 的钉子。

### D3 — luarocks 源码安装到 `~/.luarocks/` against lua@5.4

不走 brew，从 luarocks.org 下载源码，`./configure --with-lua=/opt/homebrew/opt/lua@5.4 --lua-version=5.4 --prefix=$HOME/.luarocks --rocks-tree=$HOME/.luarocks` 编译安装。

**理由**：
- 单一 rocks 树（`~/.luarocks/lib/luarocks/rocks-5.4/`），不与 brew 抢 `/opt/homebrew/` 路径。
- user-local 安装无需 sudo，跨机器复现脚本不变。
- brew 未来若放出 `luarocks@5.4` formula，可以平迁换装，不破坏 ADR 决策。

### D4 — 项目代码新增 `_resolve_busted54()`，verify_full 全 lane 走 5.4

`tools/quality/verify_full.lua` 现行 line 216-217 用裸 `busted` 字符串调用，依赖系统 PATH。新增 `_BUSTED54_BIN_CANDIDATES` + `_resolve_busted54()`，与现有 `_resolve_lua54()` 同结构，候选优先序：

```
$BUSTED54_BIN  →  $HOME/.luarocks/bin/busted  →  /opt/homebrew/bin/busted（兜底，预期不命中）
```

contract / guards lane 改为 `_resolve_busted54() .. " --run <profile>"`。

**理由**：与 coverage.lua 已有的"显式解析"模式对齐；项目不再依赖系统级 PATH 中的 `busted` 指向哪个版本，跨开发机一致。

### D5 — 唯一 rocks 树：`~/.luarocks/lib/luarocks/rocks-5.4/`

`/opt/homebrew/lib/luarocks/rocks-5.5/` 与 `/opt/homebrew/share/lua/5.5`、`/opt/homebrew/lib/lua/5.5` 物理删除。未来出现 `rocks-5.x/` 旁路视为环境异常告警。

**理由**：与 D3 配套，避免"装 rock 时不确定落到哪个树"。

### D6 — luacheck 保持 brew 安装不动

`luacheck` 1.2.0 已正确依赖 `lua@5.4`，shim 自动指 5.4 路径，是 brew 生态里干净的 keg-only 协作范例。

**理由**：能用 brew 就用 brew；不用因 D3 的源码安装路径而连带把 luacheck 也搬家。

### D7 — 不解决（明确范围外）

**N1** brew 未来若再装回 `lua`（5.5）。不加 brew tap pin 或 hook 拦截。如发生，按本 ADR 重跑 Phase 1。
**N2** CI 环境的 Lua 版本。本 ADR 仅约束本地开发机；CI 若另装 5.5 是独立问题，由 CI 配置层解决。

---

## 后果（Consequences）

**正向**：

- `lua`、`luarocks`、`busted` 三个命令全部跑在 Lua 5.4 上，与项目静态约束完全对齐。
- `tools/quality/verify_full.lua` 的 contract / guards lane 不再静默跑 5.5；CI 与本地行为收敛。
- rocks 单树，未来排查 "为什么 require 失败" 只需看一处。
- 重装机器有清晰的复现脚本（见 handoff guide）。

**代价**：

- `gnuplot` 卸了，临时需要画图时需 `brew install gnuplot`——会拽回 lua 5.5，必须先 `brew unlink lua@5.4` 才能装，装完再重新 link。
- luarocks 走源码安装，未来升级 luarocks 版本需要手动重跑源码安装步骤；brew 不再代为升级。
- `tools/quality/verify_full.lua` 改动需要重跑 verify_full 验证（toolchain 改动属于 `feedback_toolchain_verification.md` 范围）。
