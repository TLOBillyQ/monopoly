# 主回合按钮行为规格

## 背景

当前基础屏同时存在“行动按钮”和“结束按钮”，但它们的显隐和语义让玩家困惑：

- 行动按钮在可选行动阶段会被隐藏；
- 结束按钮仅在可选行动阶段出现，表示“完成当前可选行动阶段”，并非真正结束回合；
- 道具目标选择阶段没有独立的取消入口，玩家难以返回到道具槽选择。

本规格定义新的主按钮行为：

1. **行动按钮**：投骰子前显示，点击即投骰子。
2. **结束按钮**：投骰子后、且强制落地选择处理完毕后显示，点击即结束回合。
3. **取消按钮**：仅在道具目标选择阶段显示，点击返回道具槽选择。

## 目标

- 统一基础屏主按钮的语义，降低玩家认知成本。
- 将主按钮行为规格化，便于 acceptance 测试覆盖。
- 为道具目标选择阶段提供明确的取消入口。

## 非目标

- 不改变化身/皮肤/图鉴等辅助按钮。
- 不改变强制落地选择（买地、付租、机会命运等）的现有 choice 流程。
- 不新增复杂的按钮动画或特效。

## 总体行为规则

| 阶段 | 行动按钮 | 结束按钮 | 取消按钮 | 说明 |
|------|----------|----------|----------|------|
| 回合开始 ~ 投骰子前 | 显示 | 隐藏 | 隐藏 | 点击“行动”跳过 pre-action 道具，直接投骰子。 |
| 投骰子后 ~ 强制选择完成前 | 隐藏 | 隐藏 | 隐藏 | 由买地/付租/机会等强制 choice 接管界面。 |
| 强制选择完成后 ~ 回合结束 | 隐藏 | 显示 | 隐藏 | 点击“结束”跳过 post-action 道具，直接结束回合。 |
| 道具目标选择阶段 | 隐藏 | 隐藏 | 显示 | 点击“取消”返回到道具槽选择界面，不消耗道具。 |

- 倒计时超时：根据当前显示的主按钮类型，自动执行“投骰子”或“结束回合”。
- 输入锁定期间：所有主按钮均不可点击。
- 非当前玩家操作：主按钮事件被拒绝。

## 关键术语

- **行动按钮**：基础屏主按钮之一，标签为“行动”。
- **结束按钮**：基础屏主按钮之一，标签为“结束”。
- **取消按钮**：基础屏新增主按钮，标签为“取消”。
- **pre-action 道具**：投骰子前可使用的道具。
- **post-action 道具**：移动并处理完落地效果后可使用的道具。
- **强制落地选择**：玩家必须处理的落地结果，例如买地、付租、抽机会/命运卡等。
- **道具目标选择**：玩家选择某个道具后，需要进一步选择目标（如目标位置、目标玩家）的阶段。

## Gherkin 行为规格

规范源文件：`features/game/main_turn_buttons.feature`

覆盖要点：

- 回合开始时只展示行动按钮。
- 行动按钮跳过 pre-action 道具并投骰子。
- 行动按钮在目标选择阶段隐藏。
- 投骰子后强制选择期间隐藏所有主按钮。
- 强制选择完成后只展示结束按钮。
- 结束按钮跳过 post-action 道具并结束回合。
- 道具目标选择阶段只展示取消按钮。
- 取消按钮返回道具槽选择且不消耗道具。
- 倒计时超时自动投骰子 / 结束回合。
- 输入锁定期间主按钮隐藏。

## 验收标准

- [ ] 投骰子前基础屏只显示“行动”按钮，点击后投骰子。
- [ ] 投骰子后、强制落地选择完成后基础屏只显示“结束”按钮，点击后结束回合。
- [ ] 道具目标选择阶段基础屏只显示“取消”按钮，点击后返回道具槽选择。
- [ ] 倒计时超时时根据当前主按钮自动执行对应操作。
- [ ] 现有 `make test` 与 `make acceptance` 全部通过。

## 实现提示（非约束，供 coder 参考）

- 可考虑在 turn 层新增一个模块集中判断“当前该显示哪个主按钮”，避免 UI 层重复实现 turn-flow 规则。
- 行动按钮的投骰行为可能与现有 `ui_button id="next"` 的推进逻辑等价；结束按钮的结束回合行为可能需要复用或扩展 `optional_action_completion`。
- 取消按钮需要能够关闭当前道具目标选择 choice 并重新打开道具槽选择界面，而不是直接结束整个道具阶段。
- 新增取消按钮需要在 UI schema 中注册对应节点，并在输入锁定时一并禁用。
