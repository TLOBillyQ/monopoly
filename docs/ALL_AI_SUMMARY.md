# 全AI模式实现总结

## 需求
创建一个新分支：
1. 所有玩家都是AI
2. 除了没有阻断性的ui选择外，所有功能都相同

## 解决方案

### 实现方式：环境变量控制

```
┌─────────────────────────────────────────────┐
│           main.lua 启动分支逻辑              │
├─────────────────────────────────────────────┤
│                                             │
│  ALL_AI=1? ──┐                             │
│              │                             │
│         YES  │  NO                         │
│              ↓  ↓                          │
│         ┌────────┐  ┌──────────────┐      │
│         │全AI模式│  │标准UI模式    │      │
│         └────────┘  └──────────────┘      │
│              │           │                 │
│              ↓           ↓                 │
│      4个AI玩家      1人类 + 3AI           │
│      无UI层        有UI层(LoveLayer)      │
│      自动运行      等待用户输入            │
│      控制台输出    图形界面               │
│                                             │
└─────────────────────────────────────────────┘
```

### 关键机制

#### 1. AI决策自动化
```lua
-- 玩家配置（全AI模式）
{
  players = { "AI1", "AI2", "AI3", "AI4" },
  ai = { [1] = true, [2] = true, [3] = true, [4] = true },
  auto_all = true
}
```

#### 2. UI层隔离
```lua
-- 全AI模式
game.ui_port = nil  -- 不加载LoveLayer

-- 标准模式
game.ui_port = LoveLayer实例  -- 提供UI接口
```

#### 3. 选择处理流程
```
需要选择
    ↓
Agent.auto_action_for_choice()  ← AI决策引擎
    ↓
    ├─ 有决策 → 执行
    └─ 无决策 → ui_port检查
                    ↓
                ui_port == nil?
                    ↓
                ├─ 是 → 自动选择第一项/取消
                └─ 否 → 等待用户输入
```
