# Gameplay 术语对照表（以设计案为准）

目标：把代码里的术语统一到“设计案”语境，避免 `map/tile/land/landing/chance/choice` 混用造成误解。

## 核心对象

- **Board（棋盘）**：一局游戏的路线拓扑（顺序路径 + 岔路/分支）。
  - 代码：`game.board`（[src/core/board.lua](../../src/core/board.lua)）
- **Tile（格子/地块）**：棋盘上的一个位置（设计案“地块/格子”）。
  - 代码：`Tile`（[src/core/tile.lua](../../src/core/tile.lua)）
- **Property（可购买地块）**：可购买/加盖/收租的那类 Tile（`tile.type == "land"`）。
  - 设计案：地块（可购买、加盖、收租）。
  - 代码：属性效果集合（[src/gameplay/domain/land.lua](../../src/gameplay/domain/land.lua)）。

## 行为/流程

- **Move（移动）**：掷骰后沿棋盘前进（可经过起点、黑市，可能被路障打断）。
  - 代码：MovementService
- **Landing（落点/停留结算）**：移动结束后，按落点 Tile 类型触发事件（机会、道具、税务局、医院、深山、地块购买/加盖/收租等）。
  - 代码：LandingResolver + LandingEffects
- **Choice（选择）**：需要玩家输入的交互（买/不买、加盖/不加盖、道具目标等）。
  - 代码：Choice / ChoiceResolver

## 卡牌

- **Chance Card（机会卡）**：走到机会格抽取并结算。
  - 代码：ChanceService / ChanceEffects
- **Item Card（道具卡）**：走到道具格抽取并进入道具栏，或玩家主动使用。

## 索引命名（避免 tile id / 棋盘序号混淆）

- **tile.id**：配置表里的格子唯一 id（例如 35, 1, 2 ...）。
- **board_index**：在 `board.path` 中的序号（1..N），玩家位置 `player.position` 也是这个序号。

约定：在需要表达“玩家当前位置序号/落点序号”时，统一使用 `board_index`。
