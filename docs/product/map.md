## 棋盘路径结构

### 外圈（逆时针顺序，共 32 格）

```
35 → 1 → 2 → 3 → 42 → 4 → 5 → 6 → 36
→ 7 → 8 → 9 → 40 → 10 → 11 → 12 → 37
→ 13 → 14 → 15 → 41 → 16 → 17 → 18 → 38
→ 19 → 20 → 21 → 43 → 22 → 23 → 24
→ (回到 35)
```

外圈为闭合回路，逆时针行进。

### 内圈（两条相交支路，共 13 格，汇聚于黑市 39）

**纵轴**（从外圈底部入口 `42` 向上，贯穿黑市，抵达外圈顶部 `41`）：

```
42 → 45 → 31 → 32 → [39 黑市] → 28 → 29 → 30 → 41
```

**横轴**（从外圈左侧入口 `40` 向右，贯穿黑市，抵达外圈右侧 `43`）：

```
40 → 25 → 26 → 27 → [39 黑市] → 44 → 33 → 34 → 43
```

两条轴线在黑市 `39` 处汇合。玩家进入内圈后保持直线前进，穿过黑市后从对侧出口回到外圈。内圈末端（`41` / `43`）是外圈格，到达后自动回到外圈逆时针路线继续前进。

## 移动规则

### 朝向（facing / move_dir）

- `move_dir` 记录玩家**当前落点处**的下一步前进方向，是上一次移动结束时写入的航向。
- 在实现层，步进 helper 返回的朝向也遵循同一语义：表示**落点后的下一步前进方向**，不是刚刚走过来的那一步方向。
- **新鲜移动**（每回合正常掷骰）：若当前落点在外圈，以 `nil` 朝向起步，外圈第一步方向由 `outer_next`（逆时针）自动确定；若当前落点在内圈，则继续使用已保存的 `move_dir`，沿原方向直行直到回到外圈。
- **续向移动**（黑市中断或偷窃中断后恢复）：使用中断时保存的 `facing`，而非玩家存储的 `move_dir`，确保续走轨迹与中断前一致。
- **回合结束** 时，外圈玩家的 `move_dir` 清空为 `nil`；内圈玩家保留 `move_dir`，供下一回合继续直行。
- **强制传送**（送入医院 `36` 或深山 `37`）时 `move_dir` 也清空为 `nil`。

### 外圈默认方向

外圈格在**新鲜移动**下始终沿 `outer_next`（逆时针）前进，无需关心 `move_dir` 中的旧值。

### 进入内圈（入口点规则）

当本次移动的总步数 `branch_parity` 为偶数时，玩家到达入口格就会进入内圈；奇数则继续沿外圈 `outer_next` 前进，不进内圈。

若玩家上一段移动是**从内圈出来**，并且终点恰好落在入口格（`40` / `41` / `42` / `43`），则下一回合从该入口格起步时，即使 `branch_parity` 为偶数，也要先沿外圈前进，不会立刻重新进入内圈。这个限制只对**当前起点所在的那个入口格**生效一次；一旦本回合离开该入口格，后续若再到达其他入口格，仍按正常奇偶规则判定是否进圈。

同一次移动最多进入内圈一次。也就是说，玩家一旦从外圈进入内圈，本次移动即使后续再次经过其他入口格，也不会再次折返进圈。若玩家新回合起点本来就在内圈，也视为本次移动已经处于内圈路径中，直到从出口回到外圈。

进入内圈后，移动沿当前朝向保持直线前进。这个规则跨回合保持有效，直到玩家重新回到外圈。黑市 `39` 不再根据奇偶左转或右转，而是继续沿当前方向穿越到对侧支路。

### 黑市穿越中断

当移动路径**途经**黑市（黑市不是本次的终点落格，即 step < 总步数）时，移动暂停于黑市格，向玩家展示黑市界面（购买道具等）。操作结束后，以**中断时保存的 facing 和 branch_parity** 续走剩余步数（续向移动模式）。

若黑市恰好是本次的终点格（step == 总步数），则直接落地，正常触发落地事件，不产生穿越中断。

### 主动偷窃

**偷窃卡（id=2007）** 只在行动前/行动后主动使用。移动路过其他玩家不再暂停，也不再产生偷窃续走流程。

### 路障（roadblock）

玩家移动落在路障格时，**立即停止**，不再继续剩余步数，并被扣留 1 回合（`stay_turns = 1`）。路障同时被清除。

### 后退移动

步数为负数时进入**后退模式**，沿与当前朝向相反的方向逐步后退：

- 后退方向由 `move_dir` 中存储的朝向取反得到（`opposite[facing]`）。
- 后退**不更新** `move_dir`，即后退结束后玩家存储的 `move_dir` 保持不变。
- 后退会跳过黑市中断检查（`skip_market_check`）。
- 后退移动的落地事件可标记为 `allow_optional`（可选触发效果）。

### 经过起点奖励

每次经过或落在起点 `35` 时，玩家获得 **2000 金币**奖励（`pass_start_bonus = 2000`）。若一次移动中多次经过起点，则按次数累加。

### 骰子倍数

若玩家持有「骰子倍数」状态（`pending_dice_multiplier > 1`），本次新鲜移动的总步数将在消耗该状态后乘以对应倍数再执行。

## 例子

- **例子1**：起点=海口路`3`（行9, 列6），步数=4（偶数）。
  - 步骤：`3 →(outer)→ 42`（入口，facing=左，偶数满足条件2，进圈）`→ 45 → 31 → 32`
  - 终点：天津路 `32`。

- **例子2**：起点=天津路`32`，当前移动方向向下（下一格为 `31`），步数=6（偶数）。
  - 步骤：`32 → 31 → 45 → 42 → 4 → 5 → 6`
  - 终点：澳门路 `6`。

- **例子3**：起点=南昌路`25`，当前移动方向向右，步数=12（偶数）。
  - 步骤：`25 → 26 → 27 →`（到达黑市 `39`，facing=右，step=3，总步12，step<12 触发**黑市中断**，以 facing=右、parity=12 续走）`→ 44 → 33 → 34 → 43 → 22 → 23 → 24 → 35 → 1`
  - 终点：福州路 `1`。

- **例子4**：玩家上回合停在太原路`28`，保存的 `move_dir=up`，新回合步数=4（偶数）。
  - 步骤：`28 → 29 → 30 → 41 → 16`
  - 终点：哈尔滨路 `16`。

### 机会卡跳转后的朝向

- 机会卡跳转到普通地块时，不改写玩家当前 `move_dir`。
- 机会卡跳转到黑市 `39` 时，默认按逆时针方向继续，也就是当前地图里的 `right`。
- 机会卡传送到医院 `36` 或深山 `37` 时，`move_dir` 仍清空为 `nil`。

## 位置选择与范围判断

位置选择屏、目标选择器以及基于距离的范围判断，不再按图边 BFS 距离计算，而是统一使用棋盘坐标的曼哈顿距离：

`|row_a - row_b| + |col_a - col_b| <= distance`

这意味着选择范围只由地图上的行列偏移决定，不再要求两格之间必须沿当前移动图最短连通。
