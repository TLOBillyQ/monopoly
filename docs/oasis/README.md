# Areoplane 飞行棋项目

## 项目简介

Areoplane 是一个完整的飞行棋（Aeroplane Chess）多人在线游戏项目，基于虚幻引擎（Unreal Engine）和 Lua 脚本开发。这是一个4人对战的棋类游戏，每位玩家控制4架飞机，通过掷骰子决定移动步数，目标是让所有飞机最先抵达终点。

## 🎯 核心概念

### 游戏流程
```
等待玩家 → 准备阶段 → 游戏开始 → 
回合循环(掷骰子→选飞机→移动→回合结束) → 
游戏结束 → 结算
```

### 关键特性
- **掷6点起飞**: 飞机必须掷出6点才能离开起点
- **再掷一次**: 掷到6点可以再掷一次
- **吃子规则**: 落在对手位置将对手送回起点
- **按时间排名**: 最快完成的玩家获胜

## 📊 技术亮点

1. **事件驱动架构**: 使用发布-订阅模式实现松耦合
2. **状态机模式**: 清晰的游戏流程控制
3. **RPC网络通信**: 可靠的客户端-服务器通信
4. **配置驱动**: 游戏参数高度可配置
5. **模块化设计**: 代码组织清晰，易于维护和扩展

## 🛠️ 开发建议

### 修改配置
大部分游戏参数都可以在 `Script/GameConfigs/` 目录下修改：
- 时间设置：准备时间、回合时间等
- UI配置：组件路径、显示设置
- 音频配置：音效和背景音乐

### 添加功能
1. 在 `AeroplaneChessEventDefine.lua` 定义新事件
2. 在相应的逻辑文件中触发事件
3. 在UI组件中监听和响应事件

### 调试技巧
- 使用 `print()` 或 `UE.UKismetSystemLibrary.PrintString()` 输出日志
- 在编辑器中设置多人游戏模拟（Number of Players = 4）
- 监听游戏事件以跟踪游戏状态变化

## 💡 常见问题

### 如何开始本地测试？
在 Unreal Editor 中打开 `UGCmap.umap`，点击 Play 按钮。

### 如何模拟多人游戏？
在编辑器中设置 "Number of Players" 为 4，选择 "Play as Listen Server"。

### 修改Lua代码后需要重启吗？
根据项目配置，可能需要停止游戏重新启动才能看到更改。

### 在哪里可以找到更多帮助？
查看各个文档文件中的详细说明，或阅读源代码中的注释。
