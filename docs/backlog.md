# Backlog（基于 Readme 设计与当前实现差距）

## 已实现（主要要点）
- 基础回合流：ROLL → MOVE → RESOLVE → END_TURN，自动/手动模式切换，定时自动步进。
- 玩家初始化与破产处理：起始金币、4人上限（1人手控 + 3AI）、破产重置地块并判定胜利。
- 地图事件：起点奖励、机会卡抽取与执行、道具格抽取、医院/深山停留、税务局扣税、休息格奖励、黑市自动购买（简单版）。
- 地块逻辑：购买/升级、租金结算，踩地雷送医院，路障阻挡停留。
- 道具部分：抽卡与背包上限；遥控骰子（固定6点）、骰子加倍、免费卡、免税卡、均富、流放、查税、送/请神、财神/穷神/天使；路障卡、地雷卡、清障卡、偷窃卡（自动偷取）；强征卡基础版。
- 座驾对骰子数生效：有座驾投两颗骰子，无座驾一颗。
- 起点落点奖励与经过一致，均发放 `pass_start_bonus` 全额。
- 地雷触发后强制结束回合（毁车送医并提示简易 UI）。
- 升级费用/租金公式改为设计版（费用 price×2^level，租金为最后一次加盖 50%）。
- 深山收益冻结：转账/收租时在深山玩家不再获得金币。
- 税务局免税提示：有免税卡时（非自动/AI）弹窗询问是否使用；AI/自动自动使用。
- 租金合并部分实现：同主相邻地块租金叠加。
- 统一弹窗组件：UI 模态弹窗已支持键盘/鼠标统一绘制，游戏内基础提示（确认、地雷、游戏结束、税务局）接入。
- 骰子与行动前交互：实现行动前模态选择（投骰/遥控/加倍/清障），遥控支持选1-6，自动消费对应卡。

## 未实现/待完善
- **骰子与行动前交互**：细节仍需完善（例如提示文案、清障效果展示、AI 行为、重复使用限制），但基础流程已接入模态。
- **偷窃卡交互**：应逐个经过玩家弹窗选择是否使用，并可选具体道具；当前自动对首个目标随机偷取。
- **黑市**：应弹窗展示可购列表/多币种价格与卡槽满提示；当前仅用金币自动买最便宜的一件。
- **建筑与租金公式**：相邻同主地块租金合并的完整公式/显示仍需梳理（现已叠加基础租金，尚未更新 UI）。
- **地雷后续流程**：已强制结束回合并提示，仍缺更完善的 UI/按钮流和后续交互锁定。
- **税务局与免税卡交互**：设计为到税务局时提示使用免税卡；当前仅检测 free_pass，无弹窗决定。
- **机会卡/道具扩展**：怪兽卡、导弹卡等高阶道具效果未实现；机会卡对建筑/座驾的破坏类事件缺少；AI 使用策略未落地。
- **UI/提示**：剩余弹窗（购买/升级/强征/偷窃/黑市/遥控骰子等）仍未接入新模态或缺少交互细节；当前仅部分场景使用统一模态。
- **多玩家同格偷窃顺序**：设计要求经过多个玩家逐一询问；当前仅取首个目标一次。
- **深山=监狱说明**：深山停留已等同监禁，不再需要额外监狱事件，相关 UI 仍待优化。

## 实现路线图
- **骰子与行动前交互**：在投掷前弹窗/提示遥控、加倍、清障可用；遥控允许选择点数（循环1-6）；选择后写入 `pending_dice_*` 或立即清障，覆盖键鼠输入。
- **偷窃卡交互**：移动时对每个经过玩家逐一弹窗是否使用；弹窗展示目标道具栏并可选具体道具；支持“不再询问本次移动”选项；`ItemSystem:use` 接受指定目标/道具。
- **黑市体验**：进入/经过黑市弹出商品列表，支持多币种价格、卡槽满提示与多件购买；AI 按简易策略（价格/稀有度）自动购买；保留金币不足提示。
- **建筑与租金公式**：计算租金时合并相邻同主地块；更新相关 UI/提示。
- **地雷流程**：完善 UI/按钮流和交互锁定，保持“结束回合”提示。
- **税务局与免税卡**：到税务局弹窗询问使用免税卡（或 free_pass）；AI 有卡则自动使用；无卡则正常扣税并提示。
- **高阶道具与机会卡**：实现怪兽/导弹拆建筑（选范围最高价值建筑），补齐破坏座驾/建筑类机会卡；统一道具触发时机与效果文本。
- **UI/提示层**：将购买/升级/强征/偷窃/黑市/遥控骰子等弹窗接入统一模态，完善按钮与输入流，日志与屏幕提示双通路。
- **多玩家同格偷窃顺序**：按经过顺序排队弹窗，完成一个目标后继续剩余步数及后续目标。
- **深山停留展示**：完善深山（监禁）停留的 UI/提示，若需免费停留卡则复用现有 free_pass 逻辑。
- **AI 行为**：按 README 优先级实现道具使用/购买/升级/偷窃目标选择等决策，保持自动模式与 AI 一致。
- **测试回归**：单测 Tile 费用/租金计算、道具效果（偷窃/清障/路障/地雷）、税务局免税；集成测试移动含路障/黑市/起点/医院；手测弹窗与输入流程。
