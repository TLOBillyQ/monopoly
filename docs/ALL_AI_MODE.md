# 全AI模式使用指南

## 概述

本项目支持两种运行模式：

1. **标准模式**（带UI）：玩家1为人类，其他为AI
2. **全AI模式**（无UI）：所有4个玩家都是AI，自动运行到游戏结束

## 全AI模式特性

### 功能特点
- ✅ 所有玩家都是AI
- ✅ 无阻断性UI选择
- ✅ 自动运行到游戏结束
- ✅ 控制台输出游戏进度和结果
- ✅ 所有游戏逻辑完全相同

### 实现原理
- 通过 `ALL_AI=1` 环境变量启用
- 不加载UI适配层（LoveLayer）
- `game.ui_port` 保持为 `nil`
- AI决策自动处理所有选择

## 使用方法

### 方式1：使用脚本
```bash
./run_all_ai.sh
```

### 方式2：直接运行
```bash
ALL_AI=1 lua main.lua
```

### 方式3：在代码中永久启用
编辑 `main.lua`，将第9行改为：
```lua
local all_ai_mode = true  -- 永久启用全AI模式
```

## 代码修改点

### main.lua
- 添加环境变量检测
- 条件加载UI层
- 全AI模式自动运行逻辑

### 已有机制（无需修改）
- `turn_manager.lua`: 当 `ui_port == nil` 自动处理选择
- `agent.lua`: AI决策引擎
- `composition_root.lua`: 支持 `is_ai` 和 `auto_all` 配置

## 技术细节

### AI决策流程
1. 游戏创建时所有玩家标记为 `is_ai=true` 和 `auto=true`
2. 每次需要选择时，`Agent.auto_action_for_choice()` 自动决策
3. 如果AI未处理的选择，`ui_port == nil` 逻辑兜底：
   - 选择第一个选项（如果有）
   - 或取消选择（如果允许）

### 保持功能一致性
- 所有游戏规则逻辑在 `src/gameplay/` 中
- UI层仅负责渲染和输入，不影响游戏逻辑
- 全AI模式使用完全相同的游戏引擎
