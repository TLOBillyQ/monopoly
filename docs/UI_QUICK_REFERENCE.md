# UI 快速参考指南

## 屏幕布局

### 游戏板区域（左侧，585x585像素）
- **位置**：屏幕左上角，从 (50, 50) 开始
- **网格**：9x9 地块布局
- **地块尺寸**：每个地块 65x65 像素
- **总地块数**：45个

### 玩家信息面板（右侧，560x600像素）
- **位置**：屏幕右侧，从 (650, 50) 开始
- **显示内容**：
  - 每个玩家独立卡片（115像素高）
  - 玩家颜色标识
  - 金币和总资产
  - 地块和道具数量
  - 当前回合高亮显示

### 游戏状态面板（底部，550x60像素）
- **位置**：屏幕底部，从 (50, 650) 开始
- **显示内容**：
  - 当前回合数
  - 游戏阶段
  - 当前玩家
  - 操作提示

## 颜色编码

### 玩家颜色
- **玩家1**：红色 `RGB(255, 89, 89)`
- **玩家2**：蓝色 `RGB(89, 89, 255)`
- **玩家3**：绿色 `RGB(89, 217, 89)`
- **玩家4**：黄色 `RGB(255, 230, 89)`

### 地块颜色
- **起点**：绿色 `RGB(76, 204, 76)`
- **医院**：红色 `RGB(255, 102, 102)`
- **深山**：棕色 `RGB(153, 102, 51)`
- **税务局**：黄色 `RGB(204, 204, 51)`
- **黑市**：深灰 `RGB(76, 76, 76)`
- **机会卡/道具卡**：蓝色 `RGB(128, 128, 255)`
- **无主地产**：白色 `RGB(230, 230, 230)`
- **已购地产**：根据等级变化的绿色

## UI组件

### 地块信息卡片
- **尺寸**：300x400像素
- **位置**：屏幕右侧 (900, 200)
- **显示信息**：
  - 地块名称
  - 地块类型
  - 购买价格
  - 当前拥有者
  - 地块等级
  - 当前租金
  - 升级费用

### 对话框
- **尺寸**：480x220像素
- **位置**：屏幕中央 (400, 250)
- **组件**：
  - 标题栏（蓝色背景）
  - 消息内容
  - 交互按钮（支持多个）
  - 半透明黑色背景遮罩

### 骰子动画
- **尺寸**：80x80像素
- **位置**：屏幕中央 (600, 320)
- **动画时长**：0.8秒
- **效果**：数字快速变化，带阴影

## 交互方式

### 键盘操作
- **SPACE**：推进游戏（投骰子、确认等）
- **A**：切换自动/手动模式
- **H**：显示帮助信息
- **ESC**：退出游戏

### 鼠标操作
- **点击对话框按钮**：选择选项
- **点击对话框外部**：关闭对话框
- **点击地块**：（计划中）显示地块详情
- **鼠标悬停**：（已支持）按钮高亮效果

## 游戏阶段显示

### 阶段名称映射
- `waitForRoll` → "等待投骰子"
- `rolling` → "投骰子中"
- `moving` → "移动中"
- `landed` → "着陆处理"
- `turnEnd` → "回合结束"

## 动画时间线

### 骰子动画
1. **0.0s**：玩家按下SPACE键
2. **0.0-0.8s**：骰子动画播放，数字快速变化
3. **0.8s**：显示最终结果

### 玩家移动（基础实现）
1. 移动动画速度：2.0单位/秒
2. 支持平滑过渡效果

## 文件结构

```

print("Dice rolling:", animationState.diceRolling:Now())
print("Dice value:", animationState.diceValue:Now())
```

### 测试UI组件
```lua
UI.showDialog("测试", "这是一个测试对话框", {"确定", "取消"}, function(index)
    print("选择了按钮:", index)
end)
```

## 性能考虑

- 所有UI渲染都在单帧内完成
- 使用Spoke的响应式特性，避免不必要的重绘
- 动画使用delta time实现，确保帧率独立
- 颜色和字体缓存，减少创建开销
