# 设计审查报告与路线图（基于《蛋仔策划案—大富翁》v1.01）

日期：2026-01-16  
范围：根据 `design/蛋仔策划案--大富翁.docx` + `design/*.xlsx`，核查当前代码实现的缺失与不一致，并给出后续开发路线图。

  - 医院费用 5000，停留 2 回合（`src/config/constants.lua` + `src/core/player.lua`）。
  - 深山停留 2 回合、深山期间不收租（`src/core/player.lua` + `src/gameplay/land.lua`）。
  - 税务局按现金 50% 征税（`src/config/constants.lua` + `src/gameplay/land.lua`）。

- **回合流程**：
  - 回合阶段拆分（start → roll → move → land → post → end）已存在（`src/gameplay/turn_*`）。

- **地块与建筑**：
  - 地块购买/加盖/租金、连续地块租金叠加已实现（`src/gameplay/land.lua` + `src/gameplay/land_pricing.lua`）。
  - 地块/道路坐标配置存在（`src/config/tiles.lua`）。

- **机会卡系统（基础可用）**：
  - 机会卡抽取 + 事件执行（`src/gameplay/landing.lua` + `src/gameplay/chance.lua`）。

## 二、未实现 / 实现不一致点（重点问题清单）

> 下列问题按“规则缺失 → 体验缺失 → 数据不一致”的优先级排序。

### 1) 规则缺失 / 逻辑不一致（高优）

1. **胜利条件不完整**
   - 策划：时间结束比较总资产；当前仅“剩余 1 人”结束（`src/game.lua`）。
   - 缺失：时间限制、总资产结算、多人并列胜利。

2. **回合操作超时自动确认**
   - 策划：每步操作最大等待 10 秒，超时自动确认。
   - 现状：`action_timeout_seconds` 未被使用，流程无自动确认（全局无引用）。

3. **单人模式自动补 3 个 AI**
   - 策划：单人游戏自动补 3 AI。
   - 现状：默认只创建 1 名玩家（`src/gameplay/composition_root.lua`）。

4. **路障“停留 1 回合”未实现**
   - 策划：路障强制停留 1 回合。
   - 现状：路障仅停止移动并立即进入落点处理，没有 `stay_turns`（`src/gameplay/movement_service.lua`）。

5. **地雷对“不可摧毁座驾”保护缺失**
   - 策划：不可摧毁座驾免除地雷伤害。
   - 现状：地雷触发强制摧毁座驾并送医，未检测 `indestructible`（`src/gameplay/movement_service.lua` + `src/gameplay/landing.lua`）。

6. **骰子加倍卡使用时机错误**
   - 策划：投出骰子后询问是否使用（行动中）。
   - 现状：道具使用入口在行动前/行动后，没有“投骰后”阶段；`pre_move` 未落地（`src/gameplay/turn_start.lua` + `src/gameplay/turn_post.lua`）。

7. **黑市购买座驾未提示“是否替换”**
   - 策划：已有座驾时需确认替换。
   - 现状：直接替换，无确认流程（`src/gameplay/market_service.lua`）。

8. **金币归零即破产的统一判定缺失**
   - 策划：金币归零淘汰。
   - 现状：仅在租金/税务/部分机会卡触发破产；医院扣费后现金为 0 不淘汰（`src/core/player.lua`）。

### 2) 体验 / 交互缺失（中优）

1. **行动前/行动后玩家道具使用 UI 不完整**
   - 策划：行动前、行动后可使用道具。
   - 现状：仅实现行动后统一弹窗；行动前仅 AI 自动使用（`src/gameplay/turn_start.lua` + `src/gameplay/turn_post.lua`）。

2. **道具“丢弃”操作**
   - 策划：点击道具后可使用或丢弃。
   - 现状：无“丢弃”入口与逻辑（核心流程未暴露）。

3. **卡槽已满的提示**
   - 策划：抽卡失败需提示。
   - 现状：仅日志输出，缺少 UI 提示（`src/gameplay/item_inventory.lua`）。

4. **清障卡“分叉路径”清除不符合**
   - 策划：岔路时机器人分裂清除。
   - 现状：仅沿当前朝向单路径清除 12 格（`src/gameplay/item_post_effects.lua`）。

5. **偷窃卡缺少“是否使用”提示**
   - 策划：遇到玩家时询问是否使用。
   - 现状：只要有卡就进入选择流程（`src/gameplay/item_steal.lua`）。

### 3) 表格对齐结果（XLSX vs 代码配置）

> 本节依据 `design/*.xlsx` 的字段与 `src/config/*.lua` 自动比对结果。

- **道具表（`道具表.xlsx`）**
  - 字段数值整体一致；主要差异是“使用时机”字段的命名口径。
  - 例：`骰子加倍卡` 表格写法为“骰子生效前触发”，代码为 `pre_move`；
    `免税卡` 表格写法为“税务局征税时触发”，代码为 `tax_prompt`。
  - 建议：建立“文案 → 系统枚举”的映射表，避免表格写法波动导致误判。

- **机会表（`机会表.xlsx`）**
  - “事件目标”字段命名不同：`全体玩家` vs 代码 `all`。
  - “事件类型”字段命名不同：表格使用“强制前进/强制后退”，代码用 `move_forward` / `move_backward`。
  - “强制移动”参数：表格使用坐标（如 `1,9`），代码使用 `destination_tile_id`（如 `36`）。
  - 建议：
    1) 为目标/类型建立映射；
    2) 增加“坐标 → tile_id”映射或改为在表格中直接用 tile_id。

- **座驾表（`座驾表.xlsx`）**
  - `4010/4011/4012` 的“是否不可摧毁”字段为 `TURE`（疑似拼写问题），代码为 `true`。
  - 建议：修正表格值或在导入时做容错。

- **地块/黑市/角色/常量表**
  - 关键字段与代码配置一致；未发现结构性缺失。

## 三、总体评估

- 代码结构具备可扩展性（独立服务、数据配置、流程拆分）。
- 规则缺失主要集中在：胜利条件、超时自动确认、行动前道具、座驾/地雷保护、路障停留。
- 若按策划案完整度衡量，当前处于 **“核心规则可跑，但体验与细节不足”** 的阶段。

# 路线图（含重构方案）

> 目标：在最小风险下补齐规则缺失，同时为后续版本化配置与可视化 UI 提供支撑。

## Phase 0（1-2 天）：对齐需求与测试基线

- **需求确认**：
  - 从 `design/*.xlsx` 拉取最终权重/价格/座驾配置，确认是否优先级高于 docx。
- **测试基线**：
  - 新增一组“规则回归清单”，用于验证核心规则：起点奖励、税务、住院、地雷、路障、胜利条件。

## Phase 1（3-5 天）：核心规则补齐（高优）

- **胜利条件**：
  - 增加局时/回合上限；结算总资产（现金 + 地块总价值 + 座驾估值规则需明确）。
- **超时自动确认**：
  - 在 UI/Choice 层加入倒计时与默认选项行为。
- **路障停留**：
  - 遇到路障时写入 `stay_turns = 1`。
- **座驾不可摧毁**：
  - 地雷触发流程加入 `indestructible` 保护逻辑。
- **黑市替换确认**：
  - 购买座驾时若已有座驾，弹出确认选择。

## Phase 2（4-6 天）：道具/流程体验补齐（中优）

- **行动前道具窗口**：
  - 在 `turn_start` 插入“行动前道具选择”阶段。
- **骰子加倍卡流程**：
  - 在 `turn_roll` 后增加“投骰后确认使用”步骤。
- **道具丢弃**：
  - 在 UI 选择中加入“丢弃”选项并同步库存。
- **偷窃卡交互补齐**：
  - 增加“是否使用”确认提示。

## Phase 3（5-7 天）：数据与配置统一（中低优）

- **表格驱动**：
  - 统一配置为导出自 `design/*.xlsx`，清理手写冗余。
- **机会卡数量对齐**：
  - 根据最新策划表裁剪或扩展。

## Phase 4（可选重构）：流程与效果统一

> 仅在下一阶段准备大规模迭代时启动，以避免反复修改核心流程。

- **重构目标**：
  - 将“行动前/投骰后/行动后”的道具使用统一为可配置的阶段队列。
  - 将“矿雷/路障/黑市/机会卡”等触发抽象为统一的 `EffectPipeline`。
  - 消除移动路径中与落点处理中对地雷的重复判定（减少分支与重复）。

## 四、建议的交付顺序（最小可用）

1. 胜利条件 + 超时自动确认 + 路障停留（核心玩法闭环）
2. 座驾不可摧毁 + 黑市替换确认（版本一致性）
3. 行动前/投骰后道具流程（体验与策略性）
4. 表格配置全量对齐（内容稳定后做）
